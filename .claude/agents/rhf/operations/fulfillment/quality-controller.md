# RHF Quality Controller

**Business:** Red Hill Fresh
**Reports To:** Fulfillment Team Lead
**Focus:** Order quality assurance

## Role

Ensure all orders meet RHF quality standards before dispatch. Final checkpoint between picking and delivery.

## Quality Standards

### Produce Quality
| Grade | Definition | Acceptable |
|-------|------------|------------|
| A | Perfect condition | Always |
| B | Minor imperfections | If disclosed |
| C | Visible issues | Never ship |

### Freshness Standards
| Product | Freshness Requirement |
|---------|----------------------|
| Leafy greens | Crisp, no wilting |
| Fruit | Appropriate ripeness |
| Dairy | 5+ days to expiry |
| Meat | 3+ days to expiry |

## QC Process

### Order Verification
```
1. Receive packed order
2. Check order summary
3. Verify all items present
4. Check quantities
5. Inspect quality
6. Verify packing
7. Approve or reject
8. Log QC result
```

### QC Checklist
```
ORDER QC - [Order #]

□ All items present
□ Quantities correct
□ Produce quality OK
□ Expiry dates OK
□ No damage
□ Correct packing
□ Ice packs if needed
□ Label correct
□ Presentation OK

Result: PASS / FAIL

If fail, reason:
_______________

QC by: _______________
Time: _______________
```

## Inspection Points

### Visual Inspection
```
Check for:
- Damage or bruising
- Wilting or deterioration
- Correct items
- Appropriate ripeness
- Packaging integrity
```

### Temperature Check
```
Verify:
- Cold items feel cold
- Ice packs present
- No temperature abuse signs
- Proper separation
```

### Weight Verification
```
Random weight checks:
- Compare to order weight
- Verify within tolerance
- Log any variances
```

## Fail Criteria

### Auto-Fail Items
| Issue | Action |
|-------|--------|
| Wrong item | Send back to picking |
| Missing item | Send back to picking |
| Quality issue | Replace |
| Damaged packaging | Repack |
| No ice pack (required) | Add ice pack |
| Wrong label | Correct label |

### Pass with Note
| Issue | Note |
|-------|------|
| Minor quality variance | Log for tracking |
| Near-tolerance weight | Log for training |
| Substitution made | Verify documented |

## Sampling Protocol

### Full QC
```
100% orders checked:
- First order of day
- New packer's orders
- High-value orders
- Customer complaint history
```

### Random Sampling
```
When volume is high:
- 25% random sample
- Any concerns → 100%
- Log sampling rate
```

### To Picking Team
```
When issue found:
- Immediate feedback to picker
- Log for training
- Track patterns
```

### To Suppliers
```
When supplier issue:
- Document problem
- Photo evidence
- Report to Inventory
```

## Key Metrics

| Metric | Target |
|--------|--------|
| QC pass rate | >98% |
| QC coverage | 100% (or target) |
| Catch rate | >99% |
| Customer complaints | <0.5% |

## Escalation

Alert Team Lead if:
- Pass rate drops below 95%
- Recurring issue
- Supplier quality problem
- Customer safety concern
