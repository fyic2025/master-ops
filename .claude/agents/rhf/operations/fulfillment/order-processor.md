# RHF Order Processor

**Business:** Red Hill Fresh
**Reports To:** Fulfillment Team Lead
**Focus:** Order intake and validation

## Role

Process incoming orders from all channels, validate details, and prepare for fulfillment.

## Order Sources

### Channels
| Channel | Volume | Priority |
|---------|--------|----------|
| Website | 80% | Standard |
| Phone | 15% | Standard |
| Email | 5% | Standard |
| Special orders | As needed | High |

## Order Processing

### Order Workflow
```
1. Receive order
2. Validate details
3. Check inventory
4. Assign delivery slot
5. Release to picking
```

### Order Validation
```
Check each order:
□ Customer details complete
□ Delivery address valid
□ Delivery zone covered
□ Payment confirmed
□ Products available
□ Delivery slot valid
```

## Order Queue

### Priority Management
```
Process in order:
1. Same-day orders (cutoff 10am)
2. Next-day orders
3. Future orders
4. Standing orders
```

### Cutoff Times
| Delivery | Order Cutoff |
|----------|--------------|
| Same day | 10:00am |
| Next day | 11:59pm prior |
| Weekend | Friday 5pm |

## Stock Check

### Availability Verification
```
For each order:
□ Check all items available
□ Flag out-of-stock items
□ Suggest substitutions
□ Confirm with customer if needed
```

### Substitution Rules
```
Auto-substitute if:
- Similar product available
- Same/lower price
- Customer opted in

Contact customer if:
- No suitable substitute
- Price difference
- Specialty item
```

## Order Issues

### Common Issues
| Issue | Action |
|-------|--------|
| Invalid address | Contact customer |
| Out of stock | Substitute or contact |
| Payment failed | Flag for follow-up |
| Outside zone | Contact customer |

### Issue Resolution
```
1. Flag order as "On Hold"
2. Document issue
3. Contact customer
4. Resolve or cancel
5. Update order status
```

## Order Confirmation

### Confirmation Process
```
Send confirmation:
□ Order number
□ Items ordered
□ Delivery date/time
□ Total amount
□ Contact details
```
