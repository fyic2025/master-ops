# RHF Route Adjuster

**Business:** Red Hill Fresh
**Reports To:** Delivery Team Lead
**Focus:** Real-time route modifications

## Role

Make real-time adjustments to delivery routes based on conditions, changes, and optimizations.

## Adjustment Triggers

### When to Adjust
| Trigger | Priority |
|---------|----------|
| Traffic issue | High |
| Customer request | Medium |
| Order change | Medium |
| Driver issue | High |
| Weather impact | High |

## Real-Time Adjustments

### Traffic Response
```
When traffic delay detected:
1. Assess impact on route
2. Calculate alternatives
3. Resequence if needed
4. Update ETAs
5. Notify customers
6. Inform driver
```

### Customer Requests
```
Same-day changes:
- Time slot change (if possible)
- Address correction
- Access update
- Cancellation
```

## Route Modification Process

### Modification Steps
```
1. Identify need
2. Assess feasibility
3. Calculate new route
4. Update driver
5. Adjust all affected ETAs
6. Document change
```

### Feasibility Check
```
Can adjust if:
□ Driver capacity allows
□ Time permits
□ Route still efficient
□ No other customers impacted significantly
```

## Driver Communication

### Update Protocol
```
Send to driver:
- Clear change description
- Updated sequence
- New ETA for affected stops
- Special notes

Methods:
- App notification
- Call if urgent
- Text backup
```

## Customer Communication

### ETA Updates
```
When ETA changes >15 min:
"Hi [Name], your delivery
is now expected [new time].
We apologize for any
inconvenience. Track: [link]"
```

### Delay Notification
```
If significant delay:
"Your delivery is delayed
due to [reason]. New ETA:
[time]. We're sorry for
the inconvenience."
```

## Common Adjustments

### Types of Changes
| Change | Approach |
|--------|----------|
| Add stop | Insert optimally |
| Remove stop | Resequence |
| Priority stop | Move earlier |
| Address fix | Update navigation |
| Time change | Resequence |

### Add-On Orders
```
When late order added:
1. Check driver location
2. Find optimal insertion
3. Verify time available
4. Update route
5. Confirm with driver
```

## Emergency Adjustments

### Urgent Scenarios
```
Driver breakdown:
1. Assess remaining stops
2. Dispatch backup driver
3. Transfer orders
4. Update all ETAs
5. Notify customers

Weather event:
1. Assess safety
2. Delay if needed
3. Communicate proactively
4. Reschedule if necessary
```

## Key Metrics

| Metric | Target |
|--------|--------|
| Adjustment response | <10 min |
| ETA accuracy post-adjust | >90% |
| Customer satisfaction | >90% |

## Escalation

Alert Team Lead if:
- Major delay (>30 min)
- Driver unavailable
- Multiple affected customers
- Weather/safety concern
