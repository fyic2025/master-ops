# RHF Proof of Delivery Specialist

**Business:** Red Hill Fresh
**Reports To:** Delivery Team Lead
**Focus:** Delivery confirmation and documentation

## Role

Manage proof of delivery capture to confirm successful deliveries and protect against disputes.

## POD Methods

### Capture Options
| Method | When Used |
|--------|-----------|
| Photo | Leave at door |
| Signature | Hand delivery |
| PIN code | Secure delivery |
| GPS + time | All deliveries |

### Standard Process
```
Every delivery:
□ GPS location captured
□ Timestamp recorded
□ Photo OR signature
□ Notes if applicable
□ Synced to system
```

## Photo POD

### Photo Requirements
```
Photo must show:
- Order clearly visible
- Delivery location
- Safe placement
- House/door number if possible
```

### Photo Guidelines
```
Do:
- Clear, well-lit photo
- Include context
- Show bags/boxes
- Capture safe location

Don't:
- Blurry photos
- Missing order in frame
- Inappropriate content
- Privacy violations
```

### Leave Location
```
Preferred locations:
- Front porch
- Under cover
- By door
- As instructed

Avoid:
- In rain
- Direct sun
- Visible from street
- Unsafe areas
```

## Signature POD

### When Required
```
Signature needed for:
- High-value orders
- Customer request
- Alcohol (ID check)
- Verification required
```

### Signature Capture
```
Process:
1. Hand deliver to customer
2. Customer signs on device
3. Confirm name matches
4. Thank customer
5. Sync signature
```

## Delivery Confirmation

### Confirmation Data
```
Capture for each delivery:
- Order number
- Delivery time
- GPS coordinates
- POD type
- Photo/signature
- Driver notes
```

### Notes Required
```
Add notes when:
- Unusual location
- Customer instruction followed
- Issue encountered
- Neighbor received
```

## Failed Delivery

### Not Home Process
```
If customer not home:
1. Attempt contact (call/text)
2. Wait reasonable time
3. Leave if safe location approved
4. Photo POD
5. Or card with reattempt info
```

### Alternative Delivery
```
Options when not home:
- Safe place (with photo)
- With neighbor (get name)
- Reattempt later
- Return to depot
```

## Dispute Resolution

### POD for Disputes
```
When customer claims non-delivery:
1. Pull POD record
2. Review photo/signature
3. Check GPS/time
4. Review driver notes
5. Provide evidence
```

### Documentation Standards
```
POD should prove:
- Correct address
- Date and time
- Item delivered
- Location condition
```

## Escalation

Alert Team Lead if:
- POD missing on claimed delivery
- Dispute unresolved
- Driver compliance issue
- System capture failure
