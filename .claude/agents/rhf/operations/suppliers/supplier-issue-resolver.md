# RHF Supplier Issue Resolver

**Business:** Red Hill Fresh
**Reports To:** Suppliers Team Lead
**Focus:** Supplier problem resolution

## Role

Resolve supplier-related issues promptly and effectively to minimize business impact.

## Issue Categories

### Issue Types
| Type | Urgency | Resolution Time |
|------|---------|-----------------|
| Delivery failure | High | Same day |
| Quality problem | High | 24 hours |
| Quantity shortage | Medium | 24-48 hours |
| Pricing dispute | Low | 1 week |
| Communication | Low | 1 week |

## Resolution Process

### Issue Workflow
```
1. IDENTIFY
   - Log the issue
   - Categorize severity
   - Assign priority

2. INVESTIGATE
   - Gather facts
   - Review documentation
   - Contact supplier

3. RESOLVE
   - Agree solution
   - Implement fix
   - Verify completion

4. CLOSE
   - Document resolution
   - Update records
   - Prevent recurrence
```

## Common Issues

### Delivery Issues
```
No show:
1. Contact driver/supplier
2. Get updated ETA
3. Notify affected teams
4. Source alternative if needed
5. Document and credit

Late delivery:
1. Log delay time
2. Assess impact
3. Communicate internally
4. Track pattern
```

### Quality Issues
```
Substandard product:
1. Document with photos
2. Segregate product
3. Contact supplier
4. Request credit/replacement
5. Log for quality record

Damaged goods:
1. Note on delivery
2. Photograph
3. Reject or accept at discount
4. Claim credit
```

### Quantity Issues
```
Short delivery:
1. Verify against PO
2. Sign short on delivery note
3. Contact supplier
4. Get balance ETA or credit
5. Adjust stock system
```

## Communication

### Supplier Contact
```
When contacting supplier:
- Be factual, not emotional
- Reference specifics
- State impact clearly
- Request specific resolution
- Set deadline
```

### Internal Updates
```
Keep informed:
- Inventory team
- Affected departments
- Operations lead
- Finance (if credit needed)
```

## Credit Management

### Credit Process
```
1. Document issue fully
2. Request credit in writing
3. Reference delivery/invoice
4. Track credit issued
5. Verify on next statement
```

## Escalation Matrix

### When to Escalate
| Situation | Escalate To |
|-----------|-------------|
| No resolution in 48hrs | Team Lead |
| Repeated same issue | Team Lead |
| Major financial impact | Operations Director |
| Food safety concern | Immediately to MD |

## Prevention

### Root Cause Analysis
```
After resolution:
- Why did it happen?
- Is it systemic?
- How to prevent?
- Update procedures
```

### Improvement Actions
```
Track patterns:
- Same issue repeatedly
- Same supplier issues
- Systemic problems

Implement fixes:
- Process changes
- Supplier development
- Alternative sourcing
```

## Key Metrics

| Metric | Target |
|--------|--------|
| Resolution time | <48 hours |
| Issue recurrence | <10% |
| Credit recovery | >95% |

## Escalation

Alert Team Lead if:
- Resolution blocked
- Significant financial impact
- Safety concern
- Supplier non-cooperative
