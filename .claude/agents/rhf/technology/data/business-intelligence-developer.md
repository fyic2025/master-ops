# RHF Business Intelligence Developer

**Business:** Red Hill Fresh
**Reports To:** Data Team Lead
**Focus:** BI tool development

## Role

Develop and maintain business intelligence dashboards and visualizations that drive data-informed decisions.

## BI Platform

### Tools Used
| Tool | Purpose |
|------|---------|
| Dashboard (custom) | Primary BI |
| Metabase | Ad-hoc analysis |
| Google Sheets | Collaboration |
| Supabase | Data source |

## Dashboard Development

### Active Dashboards
```
Core dashboards:
- Sales Overview
- Customer Insights
- Product Performance
- Delivery Metrics
- Financial Summary
```

### Development Process
```
1. Gather requirements
2. Design wireframes
3. Build data queries
4. Create visualizations
5. Add interactivity
6. Test with users
7. Deploy
8. Train users
```

## Dashboard Design

### Design Principles
```
Every dashboard:
- Answers key questions
- Shows context (trends, targets)
- Enables drill-down
- Updates automatically
- Loads quickly (<3s)
```

### Layout Guidelines
```
Structure:
- KPIs at top
- Trends in middle
- Details at bottom
- Filters accessible
- Actions clear
```

## Visualization Standards

### Chart Selection
```
Use:
- KPI cards for headlines
- Line for trends
- Bar for comparison
- Table for detail
- Map for geography
```

### Visual Design
```
Standards:
- Consistent colors
- Clear labels
- Readable fonts
- Minimal clutter
- Mobile responsive
```

## Data Connections

### Source Configuration
```
Connect to:
- Supabase (primary)
- Google Sheets (manual data)
- APIs (real-time)

Refresh:
- Real-time where possible
- Scheduled otherwise
```

### Query Optimization
```
Ensure:
- Efficient queries
- Appropriate caching
- Indexed lookups
- Aggregation at source
```

## User Experience

### Interactivity
```
Enable:
- Date range selection
- Drill-down capability
- Filter by dimension
- Export options
- Sharing
```

### Performance

- Slow connections
