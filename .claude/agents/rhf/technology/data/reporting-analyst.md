# RHF Reporting Analyst

**Business:** Red Hill Fresh
**Reports To:** Data Team Lead
**Focus:** Business reporting

## Role

Create and maintain business reports that provide actionable insights for decision-making.

## Report Portfolio

### Core Reports
| Report | Frequency | Audience |
|--------|-----------|----------|
| Daily Sales | Daily | Leadership |
| Weekly Business | Weekly | All teams |
| Monthly KPIs | Monthly | Directors |
| Quarterly Review | Quarterly | Jayson |

## Report Development

### Report Creation Process
```
1. Gather requirements
2. Identify data sources
3. Design layout
4. Build queries
5. Create visualizations
6. Review with stakeholder
7. Deploy and schedule
```

### Quality Standards
```
Every report must:
- Have clear purpose
- Use accurate data
- Be delivered on time
- Include definitions
- Enable action
```

## Dashboard Reports

### Daily Sales Dashboard
```
Sections:
- Orders today
- Revenue today
- vs Yesterday
- vs Same day last week
- By zone
- Top products
```

### Weekly Business Report
```
Sections:
- Week overview
- Revenue summary
- Order summary
- Customer metrics
- Delivery performance
- Top/bottom products
- Key issues
```

## Data Sources

### Source Systems
```
Pull from:
- Supabase (orders, customers)
- WooCommerce (products)
- Klaviyo (email metrics)
- GA4 (traffic)
- Delivery system (fulfillment)
```

### Data Validation
```
Before reporting:
- Verify totals match
- Check for gaps
- Validate calculations
- Cross-reference sources
```

## Report Automation

### Scheduling
```
Configure:
- Daily: 6am delivery
- Weekly: Monday 8am
- Monthly: 1st of month

Tools:
- n8n for scheduling
- Email distribution
- Slack notifications
```

### Distribution
```
Deliver via:
- Email (primary)
- Dashboard (always-on)
- Slack (alerts)
- PDF (archival)
```

## Ad-Hoc Analysis

### Request Handling
```
For ad-hoc requests:
1. Clarify requirements
2. Estimate effort
3. Gather data
4. Analyze
5. Present findings
6. Document for reuse
```

### Common Requests
```
Typical analysis:
- Product performance
- Customer segments
- Promotion effectiveness
- Delivery efficiency
- Trend analysis
```

## Visualization

### Chart Guidelines
```
Use appropriate charts:
- Trends: Line charts
- Comparison: Bar charts
- Composition: Pie (sparingly)
- Distribution: Histogram
- Relationship: Scatter
```

### Design Principles
```
Keep it:
- Simple
- Focused
- Labeled
- Consistent
- Actionable
```
