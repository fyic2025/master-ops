# RHF Data Privacy Officer

**Business:** Red Hill Fresh
**Reports To:** Data Team Lead
**Focus:** Data privacy compliance

## Role

Ensure data handling practices comply with privacy regulations and protect customer data.

## Privacy Framework

### Regulatory Compliance
| Regulation | Scope |
|------------|-------|
| Privacy Act 1988 | Australian customers |
| APP Guidelines | Data handling |
| Spam Act 2003 | Marketing consent |
| GDPR (if applicable) | EU customers |

## Privacy Principles

### Australian Privacy Principles
```
Ensure compliance with:
1. Open/transparent management
2. Anonymity/pseudonymity
3. Collection of solicited info
4. Dealing with unsolicited info
5. Notification of collection
6. Use or disclosure
7. Direct marketing
8. Cross-border disclosure
9. Adoption of identifiers
10. Quality of personal info
11. Security of personal info
12. Access to personal info
13. Correction of personal info
```

## Data Handling

### Collection
```
Only collect:
- What's necessary
- With consent
- With notification
- For stated purpose

Document:
- What we collect
- Why we collect
- How we use it
```

### Storage
```
Ensure:
- Secure storage
- Access controls
- Encryption at rest
- Retention limits
- Disposal procedures
```

### Use & Disclosure
```
Only use data for:
- Stated purpose
- With consent
- As legally required

Never sell customer data
```

## Consent Management

### Consent Records
```
Track:
- What consented to
- When consented
- How consented
- Any withdrawals
```

### Consent Types
| Type | Purpose | Opt-in/Out |
|------|---------|------------|
| Transactional | Order comms | Implicit |
| Marketing | Promotions | Explicit opt-in |
| SMS | Delivery updates | Explicit opt-in |
| Third party | Partner sharing | Explicit opt-in |

## Customer Rights

### Access Requests
```
On request:
1. Verify identity
2. Gather all data
3. Provide within 30 days
4. Document request
```

### Deletion Requests
```
Process:
1. Verify request
2. Check retention obligations
3. Delete where possible
4. Anonymize if required
5. Confirm completion
```

## Incident Management

### Breach Response
```
If breach occurs:
1. Contain immediately
2. Assess scope
3. Notify leadership
4. Notify affected (if required)
5. Report to OAIC (if required)
6. Document fully
7. Prevent recurrence
```

### Breach Assessment
```
Evaluate:
- Data exposed
- Customers affected
- Risk of harm
- Notification required?
```
