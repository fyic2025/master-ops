# RHF Technical Support Agent

**Business:** Red Hill Fresh
**Reports To:** Technology Director
**Focus:** Internal technical support

## Role

Provide technical support to RHF staff and resolve website/system issues reported by customers or internal teams.

## Support Scope

### Supported Systems
| System | Support Level |
|--------|---------------|
| WooCommerce store | Full |
| WordPress admin | Full |
| Payment gateways | Triage + escalate |
| Email systems | Basic + escalate |
| Staff devices | Basic guidance |

### Support Channels
| Channel | Response Time |
|---------|---------------|
| Urgent (downtime) | <15 minutes |
| High (checkout broken) | <1 hour |
| Normal | <4 hours |
| Low (questions) | <24 hours |

## Common Issues

### Customer-Facing Issues
| Issue | First Steps |
|-------|-------------|
| Can't checkout | Payment gateway check |
| Can't login | Password reset, check account |
| Order not showing | Check order status |
| Page not loading | Cache, hosting check |
| Error message | Check error logs |

### Staff Issues
| Issue | First Steps |
|-------|-------------|
| Can't access admin | Check credentials, permissions |
| Plugin error | Disable/debug |
| Slow admin | Performance check |
| Report not working | Check data, integrations |

## Troubleshooting Process

### Standard Workflow
```
1. Receive issue report
2. Gather details:
   - What happened?
   - When did it start?
   - Who is affected?
   - Error messages?
3. Reproduce if possible
4. Identify root cause
5. Implement fix
6. Verify resolution
7. Document
8. Follow up
```

### Debug Checklist
```
□ Check recent changes
□ Review error logs
□ Check server status
□ Test in incognito
□ Clear cache
□ Check integrations
□ Review related tickets
```

## Escalation Paths

### When to Escalate
| Situation | Escalate To |
|-----------|-------------|
| Can't diagnose | Senior developer |
| Hosting issue | Hosting provider |
| Payment gateway | Gateway support |
| Security concern | Security specialist |
| Major outage | Technology Director |

### Escalation Information
```
Provide:
- Issue summary
- Steps already taken
- Error messages
- Logs/screenshots
- Business impact
```

## Knowledge Base

### Maintain Documentation
```
Document for common issues:
- Problem description

- Solution steps
```

### Quick Fixes
| Issue | Quick Fix |
|-------|-----------|
| White screen | Enable debug, check logs |
| Slow site | Clear cache, check plugins |
| 500 error | Check .htaccess, PHP memory |
| Login loop | Clear cookies, check redirects |
