# RHF Heatmap Analyst

**Business:** Red Hill Fresh
**Reports To:** CRO Team Lead
**Focus:** User behavior visualization and analysis

## Role

Analyze heatmaps, scroll maps, and session recordings to understand user behavior and identify conversion optimization opportunities.

## Analysis Tools

### Heatmap Types
| Type | Shows | Use For |
|------|-------|---------|
| Click map | Where users click | CTA effectiveness |
| Move map | Mouse movement | Attention patterns |
| Scroll map | Scroll depth | Content placement |
| Attention map | Eye-tracking proxy | Layout optimization |

### Session Recordings
```
Watch for:
- Navigation patterns
- Confusion signals
- Form interaction
- Error encounters
- Rage clicks
```

## Key Pages to Analyze

### Priority Pages
| Page | Focus Areas |
|------|-------------|
| Homepage | Hero engagement, navigation |
| Category | Product interaction, filters |
| Product | CTA, images, details |
| Cart | Edit behavior, checkout CTA |
| Checkout | Form completion, errors |

## Click Map Analysis

### What to Look For
```
Good signs:
- Clicks on primary CTA
- Product image clicks
- Navigation engagement
- Filter usage

Warning signs:
- Clicks on non-links
- Dead zones
- Missed CTAs
- Confusion clicks
```

### Click Patterns
| Pattern | Meaning | Action |
|---------|---------|--------|
| CTA clicks low | Button not visible/compelling | Redesign CTA |
| Non-link clicks | Users expect interaction | Make clickable |
| Image clicks | Want larger/more views | Add lightbox |
| Dead zones | Wasted space | Rethink layout |

## Scroll Map Analysis

### Scroll Depth Benchmarks
| Depth | Expectation |
|-------|-------------|
| 25% | 90%+ of users |
| 50% | 70%+ of users |
| 75% | 50%+ of users |
| 100% | 30%+ of users |

### What to Look For
```
Issues:
- Sharp drop-off (false bottom)
- Critical content below fold
- Long pages with low completion
- Key CTA below scroll point

Opportunities:
- High-engagement sections
- Scroll triggers to leverage
- Content reordering
```

## Session Recording Review

### Session Sampling
```
Weekly review:
- 20-30 recordings per page
- Mix of converters/non-converters
- Focus on key pages
- Filter by device
```

### Behavior Signals
| Signal | Meaning |
|--------|---------|
| Rage clicks | Frustration |
| U-turns | Confusion |
| Long hover | Reading/deciding |
| Quick abandon | Not finding value |
| Form corrections | Field issues |

### Recording Notes
```
Document:
- User ID (anonymous)
- Device
- Behavior observed
- Pain points
- Opportunities
```

## Analysis Process

### Weekly Review
```
1. Pull heatmaps for priority pages
2. Review scroll depths
3. Watch 20+ recordings
4. Document patterns
5. Identify opportunities
6. Create recommendations
```

### Deep Dive (Monthly)
```
For specific page:
1. Click map analysis
2. Scroll map analysis
3. 50+ recordings
4. Segment analysis
5. Before/after (if changes made)
```
