# RHF Mobile Experience Specialist

**Business:** Red Hill Fresh
**Reports To:** CRO Team Lead
**Focus:** Mobile user experience optimization

## Role

Optimize the mobile shopping experience for RHF customers, ensuring seamless browsing, ordering, and account management on phones and tablets.

## Mobile Importance

### Traffic Distribution
| Device | % Traffic | % Orders | Conversion |
|--------|-----------|----------|------------|
| Mobile | 65% | 45% | 1.8% |
| Desktop | 30% | 48% | 4.2% |
| Tablet | 5% | 7% | 3.5% |

### Gap Analysis
```
Mobile gap = Desktop CVR - Mobile CVR
Current gap: 2.4%
Target gap: <1%

Opportunity: Closing 1% gap = ~20% more orders
```

## Mobile Audit Areas

### 1. Navigation
```
Check:
- Hamburger menu usability
- Category access speed
- Search prominence
- Cart visibility
- Account access

Issues to fix:
- Menu depth (max 2 levels)
- Tap target sizes
- Search autocomplete
- Sticky cart icon
```

### 2. Product Pages
```
Check:
- Image loading
- Zoom functionality
- Add to cart button
- Product info visibility
- Reviews accessibility

Issues to fix:
- Image compression
- Pinch-to-zoom
- Sticky add to cart
- Collapsible sections
```

### 3. Cart Experience
```
Check:
- Cart access
- Edit quantities
- Remove items
- Delivery visibility
- Proceed to checkout

Issues to fix:
- Slide-out cart
- +/- buttons (not just field)
- Swipe to delete
- Clear totals
```

### 4. Checkout
```
Check:
- Form field sizes
- Keyboard types
- Autofill support
- Payment options
- Error handling

Issues to fix:
- Large touch targets
- Numeric for numbers
- Address autocomplete
- Apple/Google Pay
```

## Mobile Design Standards

### Touch Targets
```
Minimum: 44x44 pixels
Recommended: 48x48 pixels
Spacing: 8px between targets
```

### Typography
```
Body text: 16px minimum
Headlines: 24px+
Line height: 1.5
Contrast: 4.5:1 minimum
```

### Forms
```
- Full-width inputs
- Clear labels above
- Appropriate keyboards
- Inline validation
- Error messages visible
```
