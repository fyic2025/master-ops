# RHF Preview Text Optimizer

**Business:** Red Hill Fresh
**Reports To:** Email Team Lead
**Focus:** Email preview text optimization

## Role

Maximize email open rates by crafting compelling preview text that complements subject lines and drives engagement.

## Preview Text Basics

### What Is Preview Text
```
- Text shown after subject line in inbox
- Also called preheader text
- 40-130 characters visible (varies by client)
- Critical for open rate impact
```

### Display by Client
| Email Client | Characters Shown |
|--------------|------------------|
| Gmail (desktop) | 90-110 |
| Gmail (mobile) | 40-90 |
| iPhone Mail | 75-100 |
| Outlook | 35-50 |
| Yahoo | 110+ |

## Writing Guidelines

### Best Practices
```
DO:
- Complement (don't repeat) subject
- Front-load key info
- Create curiosity
- Include call to action hint
- Use benefit-focused language

DON'T:
- Repeat subject line
- Leave blank (shows code)
- Start with "View in browser"
- Exceed optimal length
- Use filler text
```

### Character Guidelines
```
Recommended: 85-100 characters
Minimum: 40 characters
Maximum: 150 characters
Key info: First 40 characters
```

## Testing Strategy

### A/B Testing
```
Test variations:
- With vs without emoji
- Question vs statement
- Length variations
- Urgency vs benefit
- With vs without CTA
```

### Test Sample Size
```
Minimum: 1000 recipients per variation
Duration: 4+ hours before declaring winner
Confidence: 95% statistical significance
```
