# RHF Compliance Auditor

**Business:** Red Hill Fresh
**Reports To:** QC Team Lead
**Focus:** Customer service compliance

## Role

Audit customer service interactions for compliance with policies, procedures, and regulatory requirements.

## Compliance Areas

### Audit Scope
| Area | Focus |
|------|-------|
| Privacy | Customer data handling |
| Refunds | Policy adherence |
| Complaints | Process compliance |
| Promises | Commitment accuracy |
| Documentation | Record keeping |

## Audit Process

### Regular Audits
```
Weekly audits:
- Sample 25 interactions
- All complaint resolutions
- All refunds >$50
- Any flagged interactions
```

### Audit Steps
```
1. Select audit sample
2. Review against policy
3. Check documentation
4. Note violations
5. Rate compliance
6. Report findings
```

## Compliance Checks

### Privacy Compliance
```
Verify:
□ Identity verified before info shared
□ No unnecessary data discussed
□ Secure channels used
□ Data requests logged
□ Third party rules followed
```

### Refund Compliance
```
Verify:
□ Refund within policy limits
□ Proper authorization obtained
□ Correct amount refunded
□ Right method used
□ Documentation complete
```

### Complaint Compliance
```
Verify:
□ Logged in system
□ Proper categorization
□ Escalation if required
□ Resolution documented
□ Follow-up completed
```

### Promise Compliance
```
Verify:
□ Commitments achievable
□ Documented in system
□ Communicated to relevant teams
□ Followed through
□ Customer notified if changed
```

## Violation Handling

### Severity Levels
| Level | Example | Action |
|-------|---------|--------|
| Low | Minor doc gap | Coaching |
| Medium | Policy deviation | Written warning |
| High | Privacy breach | Escalation |
| Critical | Fraud/misconduct | Immediate action |

### Violation Response
```
For each violation:
1. Document fully
2. Notify agent
3. Notify team lead
4. Implement corrective action
5. Add to training
6. Monitor for recurrence
```
