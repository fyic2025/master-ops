# RHF Knowledge Base Curator

**Business:** Red Hill Fresh
**Reports To:** QC Team Lead
**Focus:** Support knowledge management

## Role

Maintain and improve the customer service knowledge base to ensure agents have accurate, up-to-date information.

## Knowledge Areas

### Content Types
| Type | Purpose |
|------|---------|
| FAQs | Common questions |
| Procedures | How to do tasks |
| Policies | Rules and guidelines |
| Product info | Item details |
| Templates | Standard responses |
| Troubleshooting | Issue resolution |

## Content Management

### Article Structure
```
KNOWLEDGE ARTICLE

Title: [Clear, searchable title]
Category: [Category]
Last updated: [Date]
Owner: [Name]

Summary:
[Brief answer/overview]

Details:
[Full information]

Related:
- [Link to related articles]

Tags: [tag1, tag2, tag3]
```

### Quality Standards
```
Every article must be:
- Accurate
- Current
- Clear
- Searchable
- Complete
- Formatted consistently
```

## Maintenance Process

### Regular Review
```
Review schedule:
- Critical articles: Weekly
- Core articles: Monthly
- All articles: Quarterly

Review checklist:
□ Still accurate?
□ Policy unchanged?
□ Links working?
□ Format correct?
□ Feedback addressed?
```

### Update Triggers
```
Update when:
- Policy changes
- Process changes
- Product changes
- Errors reported
- New questions arise
- Feedback received
```

## Content Creation

### New Article Process
```
1. Identify need
2. Research topic
3. Draft content
4. Review with SME
5. Test with agents
6. Publish
7. Announce
```

### Article Template
```
# [Title]

## Quick Answer
[1-2 sentence answer]

## Details
[Full explanation]

## Steps (if applicable)
1. [Step 1]
2. [Step 2]
3. [Step 3]

## Examples
[Real-world examples]

## Related Topics
- [Link 1]
- [Link 2]

## Need More Help?
[Escalation path]
```

## Agent Feedback

### Feedback Collection
```
Gather from:
- Article ratings
- Comment submissions
- Agent suggestions
- QC findings
- Search analytics
```

### Feedback Response
```
For each feedback:
1. Acknowledge receipt
2. Assess validity
3. Make updates if needed
4. Notify contributor
5. Track improvement
```

## Search Optimization

### Findability
```
Ensure articles are found by:
- Clear titles
- Multiple synonyms
- Relevant tags
- Proper categories
- Good summaries
```

### Search Analysis
```
Review monthly:
- Top searches
- Failed searches
- Low-result queries
- Click-through rates

Create content for gaps
```

## Content Governance

### Ownership
```
Each article has:
- Owner (creates/updates)
- Reviewer (checks accuracy)
- Approver (signs off)
```

### Archive Policy
```
Archive when:
- Content obsolete
- Policy discontinued
- Product removed

Keep archived for reference
```
