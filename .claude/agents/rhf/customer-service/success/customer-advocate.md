# RHF Customer Advocate

**Business:** Red Hill Fresh
**Reports To:** Success Team Lead
**Focus:** Customer voice representation

## Role

Represent the customer's perspective internally, ensuring their needs and feedback drive business decisions.

## Advocacy Areas

### Key Responsibilities
| Area | Action |
|------|--------|
| Feedback synthesis | Compile insights |
| Issue patterns | Identify trends |
| Product requests | Champion needs |
| Process improvement | Recommend changes |
| Experience gaps | Highlight issues |

## Voice of Customer

### Feedback Collection
```
Aggregate from:
- Support tickets
- NPS surveys
- Social media
- Reviews
- Direct feedback
- Exit surveys
```

### Theme Analysis
```
Weekly analysis:
1. Categorize all feedback
2. Identify patterns
3. Quantify impact
4. Prioritize themes
5. Present to teams
```

## Internal Advocacy

### Team Presentations
```
Monthly customer voice meeting:

"Here's what customers told us:

Top praise:
1. [Theme] - [count] mentions
2. [Theme] - [count] mentions

Top concerns:
1. [Theme] - [count] mentions
   Impact: [description]
   Suggested action: [action]

Customer quote:
'[Actual quote]'"
```

### Department Briefs
```
For each department:
- Relevant customer feedback
- Specific pain points
- Suggested improvements
- Success stories
```

## Issue Escalation

### Systemic Issues
```
When pattern identified:
1. Document issue
2. Quantify affected customers
3. Calculate impact ($, satisfaction)
4. Propose solution
5. Present to leadership
6. Track resolution
```

### Issue Report
```
CUSTOMER ISSUE BRIEF

Issue: [Description]
Affected customers: [count]
Frequency: [increasing/stable]
Impact: [Revenue/Satisfaction]

Customer quotes:
- "[Quote 1]"
- "[Quote 2]"

Proposed solution: [Action]
Resource needed: [What]
Expected outcome: [Result]
```

## Product Advocacy

### Feature Requests
```
Track and prioritize:
- Request volume
- Revenue potential
- Implementation effort
- Customer impact

Present quarterly:
"Top requested features by customers"
```

### New Product Input
```
Before launch:
- Customer research
- Pain point alignment
- Pricing feedback
- Naming preferences
- Feature priorities
```

- Major customer experience gap
- Competitive threat identified
- Significant revenue risk
- Opportunity for innovation
