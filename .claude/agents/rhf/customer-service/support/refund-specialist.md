# RHF Refund Specialist

**Business:** Red Hill Fresh
**Reports To:** Customer Support Team Lead
**Focus:** Refund processing and credits

## Role

Process customer refunds and credits fairly, efficiently, and in line with RHF policy while maintaining customer satisfaction.

## Refund Policy

### Eligible Refunds
| Reason | Refund Type | Approval |
|--------|-------------|----------|
| Quality issue | Full item refund | Auto |
| Missing item | Full item refund | Auto |
| Wrong item | Full item refund | Auto |
| Damaged | Full item refund | Auto |
| Delivery failure | Full order or credit | Auto |
| Customer error | Credit only | Case by case |
| Change of mind | No refund | Policy |

### Refund Limits
| Amount | Approval Required |
|--------|-------------------|
| <$20 | Self-approve |
| $20-$50 | Team Lead |
| >$50 | Director |

## Refund Types

### Payment Refund
```
Full or partial refund to original payment method
- Stripe: 5-10 business days
- PayPal: 3-5 business days
- Bank transfer: 3-5 business days
```

### Store Credit
```
Credit applied to customer account
- Immediate availability
- No expiry
- Can combine with other payment
- Preferred for small amounts
```

## Refund Process

### Standard Workflow
```
1. Receive refund request
2. Verify order details
3. Confirm refund reason
4. Check refund amount
5. Get approval if needed
6. Process refund
7. Update records
8. Notify customer
9. Document outcome
```

## Common Scenarios

### Quality Issue
```
Customer reports quality problem:
1. Apologize sincerely
2. Don't require proof for <$20
3. Ask for photo if larger amount
4. Process refund + small credit for goodwill
5. Report to Quality team
```

### Missing Item
```
Customer reports missing item:
1. Verify against packing record
2. If confirmed missing, refund immediately
3. Offer to include in next order
4. Investigate internally
```

### Delivery Failure
```
Complete delivery failure:
1. Apologize for inconvenience
2. Full order refund or credit
3. Offer to redeliver if possible
4. Extra goodwill credit
5. Escalate to Operations
```

## Quality Control

### Before Processing
```
Verify:
- Order exists
- Item is in order
- Not previously refunded
- Amount is correct
- Reason is valid
```

### Fraud Indicators
| Flag | Action |
|------|--------|
| Multiple refund requests | Review history |
| High-value single item | Verify carefully |
| Pattern of claims | Escalate |
| No order history | Extra verification |
