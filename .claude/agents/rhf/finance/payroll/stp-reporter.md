# RHF STP Reporter

**Business:** Red Hill Fresh
**Reports To:** Payroll Team Lead
**Focus:** Single Touch Payroll reporting

## Role

Ensure timely and accurate Single Touch Payroll (STP) reporting to the ATO after each pay run.

## STP Basics

### What is STP
```
Single Touch Payroll:
- Real-time payroll reporting
- Sent to ATO each pay day
- Replaces payment summaries
- Employee can view via myGov
```

### Reporting Obligations
| Element | Requirement |
|---------|-------------|
| Frequency | Each pay event |
| Timing | On or before pay day |
| Method | Via payroll software |
| Finalisation | 14 July annually |

## STP Data

### Per Pay Event
```
Report for each employee:
- Gross payments
- PAYG withholding
- Super liability
- Reportable FB
- Allowances
- Deductions
```

### Year-to-Date
```
Running totals:
- Total gross
- Total PAYG withheld
- Total super
- Total reportable amounts
```

## Reporting Process

### Each Pay Run
```
1. Complete pay calculations
2. Verify data accuracy
3. Submit STP report
4. Receive confirmation
5. Save ATO receipt
6. Resolve any errors
```

### Submission Steps
```
In payroll software:
1. Review STP summary
2. Check for warnings
3. Click "Submit to ATO"
4. Wait for response
5. Download confirmation
```

## Error Handling

### Common Errors
| Error | Resolution |
|-------|------------|
| Invalid TFN | Verify with employee |
| Name mismatch | Check ATO records |
| Rejection | Review and resubmit |

### Resolution Process
```
1. Note error message
2. Identify cause
3. Correct data
4. Resubmit
5. Verify acceptance
```

## Finalisation

### Annual Finalisation
```
By 14 July:
- Make STP final
- Mark as "Final" for year
- Submit finalisation
- Confirm ATO receipt
```

### Finalisation Checklist
```
Before finalising:
□ All pays processed
□ Corrections made
□ Terminations final
□ RFBA included
□ Ready for year-end
```
