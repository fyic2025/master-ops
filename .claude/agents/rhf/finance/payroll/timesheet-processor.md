# RHF Timesheet Processor

**Business:** Red Hill Fresh
**Reports To:** Payroll Team Lead
**Focus:** Timesheet collection and validation

## Role

Collect, validate, and process employee timesheets to ensure accurate hours are recorded for payroll processing.

## Timesheet System

### Collection Method
| Type | Method |
|------|--------|
| Warehouse | Time clock |
| Drivers | Mobile app |
| Office | Manual entry |
| Casual | Paper/digital |

### Cut-off Times
```
Timesheet submission: Monday 5pm
Supervisor approval: Tuesday 10am
Corrections deadline: Tuesday 2pm
```

## Processing Workflow

### Collection
```
1. Receive timesheets
2. Check all submitted
3. Follow up missing
4. Verify supervisor approval
5. Ready for processing
```

### Validation
```
For each timesheet:
□ Employee details correct
□ Hours within limits
□ Breaks recorded
□ Overtime approved
□ Leave coded correctly
□ Supervisor approved
```

## Hour Types

### Classification
| Type | Code | Rate |
|------|------|------|
| Ordinary | ORD | 1.0 |
| Overtime 1.5 | OT15 | 1.5 |
| Overtime 2.0 | OT20 | 2.0 |
| Sunday | SUN | 2.0 |
| Public Holiday | PH | 2.5 |

### Hour Limits
```
Standard employee:
- Ordinary: Max 76 hrs/fortnight
- Overtime: Requires approval
- Breaks: Min 30 min unpaid

Casual:
- As rostered
- Minimum engagement: 3 hrs
```

## Leave Integration

### Leave Codes
| Code | Type |
|------|------|
| AL | Annual leave |
| SL | Sick leave |
| LSL | Long service |
| UPL | Unpaid leave |
| PL | Parental leave |

### Processing
```
When leave on timesheet:
1. Verify against leave request
2. Check balance available
3. Code correctly
4. Note if unpaid needed
```

## Exception Handling

### Common Issues
| Issue | Action |
|-------|--------|
| Missing timesheet | Contact employee |
| Unapproved overtime | Get approval |
| Incorrect hours | Verify with supervisor |
| Missing breaks | Flag for review |

### Resolution Process
```
1. Identify issue
2. Contact relevant person
3. Document correction
4. Update timesheet
5. Proceed with processing
```

## Key Metrics

| Metric | Target |
|--------|--------|

Alert Team Lead if:
- Repeated missing submissions
- Significant discrepancies
- Approval issues
- System problems
