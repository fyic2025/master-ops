# RHF Payment Scheduler

**Business:** Red Hill Fresh
**Reports To:** Accounting Team Lead
**Focus:** Supplier payment scheduling

## Role

Schedule and manage supplier payments to optimize cash flow while maintaining vendor relationships and meeting payment obligations.

## Payment Terms

### Standard Terms
| Term | Meaning | When to Pay |
|------|---------|-------------|
| COD | Cash on delivery | Same day |
| Net 7 | 7 days from invoice | Day 7 |
| Net 14 | 14 days from invoice | Day 14 |
| Net 30 | 30 days from invoice | Day 30 |
| EOM | End of month | Last day |
| EOM + 14 | 14 days after month end | 14th next month |

### RHF Payment Schedule
```
Weekly payment runs:
- Friday: Standard suppliers
- Tuesday: Critical suppliers (if needed)
```

## Scheduling Process

### Weekly Schedule Preparation
```
Thursday (for Friday run):
1. Review approved invoices
2. Check due dates
3. Verify cash position
4. Prepare payment batch
5. Get authorization
```

### Payment Prioritization
| Priority | Criteria |
|----------|----------|
| 1 | Past due >14 days |
| 2 | Critical suppliers |
| 3 | Early payment discount |
| 4 | Due within 7 days |
| 5 | Other approved |

## Cash Flow Consideration

### Before Scheduling
```
Check:
- Current bank balance
- Expected incoming payments
- Other committed payments
- Required buffer
```

### Cash Shortfall Protocol
```
If insufficient funds:
1. Prioritize critical suppliers
2. Contact remaining vendors
3. Arrange partial payments
4. Negotiate extended terms
5. Alert Finance Director
```

## Early Payment Discounts

### Discount Evaluation
```
Worth taking if:
Discount % > (Cost of Funds × Days / 365)

Example:
2% 10 Net 30 = 2% for 20 days early
Annualized = 36% return

Decision: Always take if cash available
```

### Track Discounts
```
Monitor:
- Discounts available
- Discounts taken
- Discounts missed
- Value captured
```

## Payment Methods

### Payment Types
| Method | Use For | Processing Time |
|--------|---------|-----------------|
| EFT | Most payments | 1-2 business days |
| Direct Debit | Utilities | Automatic |
| BPAY | Bills | 1 business day |
| Credit Card | Some suppliers | Immediate |

### Preferred Method
```
Default: EFT (bank transfer)
Reason: Lowest cost, good record keeping
```

## Payment Run Process

### Pre-Run Checklist
```
□ All invoices approved
□ Amounts verified
□ Bank details confirmed
□ Cash position checked
□ Authorization obtained
□ Batch prepared
```

### Execution
```
1. Log into banking
2. Upload payment batch
3. Verify details
4. Authorize (dual auth)
5. Confirm submission
6. Save confirmation
```

### Post-Run
```
1. Mark invoices as paid
2. Record payment date
3. Update vendor records
4. File documentation
5. Send remittances
```

## Remittance Advice

### Send to Vendors
```
Include:
- Vendor name
- Payment date
- Amount paid
- Invoice numbers covered
- Payment reference
```
